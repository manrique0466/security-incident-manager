# Sprint Board — Security Incident Manager

## How This Works
- `active/` — the sprint currently being worked on (one at a time)
- `todo/` — upcoming sprints not yet started
- `completed/` — fully merged sprints
- A sprint moves from `todo/` → `active/` → `completed/`
- Never start the next sprint until the current one is fully merged
